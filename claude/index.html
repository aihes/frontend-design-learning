01 / Philosophy

Claude 的六个设计信念

每个信念都来自一个真实的设计判断——不是从教科书抄来的规则，而是"这样做更好"的经验。

🎨

视觉层级即信息架构

用户扫描页面的顺序就是你的优先级列表。字号、粗细、间距的每一级变化都是在说"先看这里，再看那里"。

⛰

留白是设计，不是空缺

好的留白创造呼吸感和节奏。它不是"还没填满的地方"，而是主动选择——让重要的东西被看见。

📖

系统先于页面

先建立色彩 token、字体比例、间距变量、组件模式，再用这些积木拼页面。没有系统就没有一致性。

⚙

动效服务于理解

动画不是装饰。hover 反馈告诉用户"这里可交互"，页面转场告诉用户"你从哪来、到哪去"。没有意图的动效是噪音。

📏

约束激发创造力

限制字体种类、只用 3-5 个颜色、固定间距比例。约束越明确，设计越一致，也越容易做出"看起来专业"的结果。

🧭

设计要可解释

每个设计决策都应该能用一句话解释"为什么"。如果解释不出来，这个设计可能只是惯性或审美偏好，不是判断。

02 / Typography

排版是设计的骨架

在还没有颜色和图片之前，光靠字号的层级关系就能让页面清晰可读。

2.1

字体比例系统

用一个一致的字号比例（scale）来建立自然的视觉层级

推荐做法 · Type Scale
GOOD
display 让每个像素都有意义
h1 清晰的主标题层级
h2 二级标题引导视线
body 正文使用舒适的行高和字重，确保长文可读性。16px 是最安全的起始值。
caption 辅助信息使用较小字号和低对比度
常见问题
AVOID
??? 标题不够突出
??? 跟上面的区分不大
??? 还是分不清层级
??? 正文太小太密，读起来很累，行高也不够。
??? 这个太小了根本看不清
💡

关键比例：每级标题之间保持 1.2–1.5 倍的字号差距。用 font-weight 的跳跃（400 → 700）来强化层级，而不是只靠字号微调。line-height 随字号增大而减小：正文 1.7，标题 1.1。

2.2

字体配对原则

衬线 + 无衬线的经典组合，建立明确的角色分工

Font Pairing
SERIF — 标题用
Crimson Pro
优雅的曲线，高辨识度
SANS — 正文用
Inter 是为屏幕设计的无衬线字体。
在小尺寸下依然清晰，中英文混排效果好。
03 / Color

色彩不是感觉，是系统

用 CSS 变量建立色彩 token，每个颜色都有名字、有角色、有使用规则。

3.1

Claude 的色彩 token 体系

语义化命名，不是"蓝色"而是"主强调色"

--ink
#1a1714 · 主文字
--muted
#8a8279 · 次文字
--dim
#b5afa6 · 装饰
--accent
#d97706 · 强调
--teal
#0f766e · 信息
--emerald
#059669 · 成功
--rose
#be123c · 警示
--bg
#faf8f5 · 底色
💡

核心规则：一个页面最多 1 个主强调色 + 2-3 个语义色（info / success / error）。底色用暖白而非纯白（#fff 太冷），文字色用暖黑而非纯黑（#000 太硬）。

3.2

对比度不是数字，是可读性

WCAG AA 是底线，不是目标

Good Contrast
AA +
深色文字在暖白底上
对比度 12:1+，长时间阅读不累。正文用 --ink-secondary (#3d3832) 也有 8:1+。
深色背景 + 亮色文字 — 同样高对比度
Poor Contrast
FAIL
浅灰文字在白底上
对比度只有 2:1，根本看不清。这种"优雅的低对比度"只是对视力好的设计师优雅。
深底 + 灰字 — 同样的问题
04 / Spacing

间距是看不见的网格

用 4px 或 8px 的倍数来组织所有空间，创造自然的节奏感。

4.1

8px 基础间距系统

所有间距都是 8 的倍数，半步用 4

Spacing Scale
4px
icon gap, inline spacing
8px
tight gaps, tag padding
12px
card internal, small sections
16px
card padding, grid gap
24px
section internal, content blocks
32px
card padding, hero spacing
48px
section gaps, major rhythm
80px
page sections, hero padding
💡

核心法则：元素内部间距 < 外部间距。同一模块内用 12-20px，模块之间用 32-48px，大区块用 80px。这种"内紧外松"创造自然的分组感。

05 / Layout

布局是页面的逻辑地图

好的布局让用户不需要思考"这个东西在哪"，因为位置本身就在传达含义。

5.1

弹性布局模式

用 CSS Grid 和 Flexbox 构建可预测的、响应式的布局

3-Column Grid
nav
brand
cta
hero — full width
card 1
card 2
card 3

三等分适合并列展示，等权重内容。

Asymmetric Layout
main content
占据 2/3
sidebar
占 1/3
feature
feature

不对称布局建立主次关系，引导注意力。

5.2

容器策略

max-width + padding，不依赖固定宽度

Container Pattern
.container {
  max-width: 1120px;
  margin: 0 auto;
  padding: 0 clamp(20px, 4vw, 48px);
}

clamp() 让 padding 在小屏幕上自动缩小，大屏幕上自动放大——不需要断点查询。

08 / Process

Claude 做前端设计的四步流程

不是"画页面"，而是先思考、再系统、然后组装、最后打磨。

01

理解意图

这个页面要说什么？用户 5 秒内应该获得什么信息？先想清楚叙事，再决定视觉。

→
02

建立系统

定义 CSS 变量：色彩 token、字体比例、间距基准、圆角和阴影层级。这是设计语言的字典。

→
03

组装页面

用系统里的积木搭页面：容器 → 区域 → 组件。每一步都从系统里取值，不临时发明。

→
04

打磨体验

加 hover 状态、过渡动画、响应式适配、可访问性检查。这是从"能用"到"好用"的一步。
