# interactive-portfolio

## 适用场景

用于个人网站、作品集、简历站、候选人主页。核心不是把简历字段排成网页，而是让访问者在很短时间内判断：

- 这个人是谁？
- 他能解决什么问题？
- 最强证明是什么？
- 下一步如何联系或继续了解？

## 输入

- 目标岗位或目标机会。
- 原始简历、项目列表、作品链接。
- 可公开和不可公开的信息边界。
- 候选人最想传达的定位。
- 已验证的项目结果、数据和截图。

## 输出

- 候选人一句话定位。
- 首屏内容结构。
- 项目证明排序。
- 每个项目的问题、角色、动作、结果。
- 联系或下一步 CTA。
- 隐私处理清单。

## 工作流

1. 先做 30 秒测试。
   - 30 秒内必须看懂身份、方向、最强作品和下一步。
   - 如果首屏只是在复述学历或职位，说明还没有完成叙事重组。

2. 把经历改成证明顺序。
   - 不按时间线平均展示所有经历。
   - 优先展示最能证明目标岗位能力的 2-4 个项目。
   - 背景经历只用于补强可信度。

3. 每个项目按证据链展开。
   - Problem：用户或业务问题是什么？
   - Role：你负责什么？
   - Decision：关键判断是什么？
   - Execution：怎么落地？
   - Signal：有什么结果或反馈？

4. 控制可信边界。
   - 不编造指标。
   - 不把团队成果写成个人独占。
   - 不暴露邮箱、手机号、内部链接、未授权客户信息。

## 页面结构模板

```text
Hero
- 姓名或候选人定位
- 目标机会相关的一句话
- 2-3 个最强信号
- CTA

Selected Work
- 项目 1：最强证明
- 项目 2：第二强证明
- 项目 3：补充证明

Capabilities
- 3 个和目标岗位直接相关的能力

Background
- 教育、实习、经历，只保留能支撑判断的信息

Contact / Next Step
- 不泄露隐私的联系入口或下一步说明
```

## 验收标准

- 首屏不是“简历网页化”，而是“机会判断页”。
- 项目证明比经历堆叠更突出。
- 每个项目都有真实动作和结果信号。
- 不靠装饰压过内容。
- 移动端也能快速扫读。
