# landing-page-design

## 适用场景

用于产品页、SaaS landing page、活动页、工具页和任何需要转化路径的页面。

## 输入

- 目标用户。
- 产品价值。
- 转化目标。
- 信任证据。
- 竞品或参考页。
- 可公开的截图、数据和素材。

## 输出

- 5 秒首屏。
- 标题和副标题。
- CTA 文案。
- 信任证据位置。
- section 顺序。
- 移动端首屏优先级。

## 首屏公式

```text
Logo / Nav / CTA

Headline：6-12 个词，直接说结果
Subheadline：解释怎么做到
Hero visual：展示结果或使用场景
Primary CTA：动作 + 价值
Trust proof：logo、数字、短证言或真实案例
```

## 标题原则

不要写：

- Welcome to our platform
- Next generation solution
- Seamless experience
- We help you grow

改成：

- 说明用户得到什么结果。
- 说明节省什么时间或减少什么痛点。
- 使用具体动词，而不是营销空话。

## Section 顺序

```text
Hero
Problem / Stakes
Solution
Proof
How it works
Use cases
Pricing or CTA
FAQ
Final CTA
```

## 验收标准

- 5 秒内知道产品是什么、给谁、有什么价值。
- CTA 不是孤立按钮，而是承接上下文。
- 信任证据真实可验证。
- hero image 不只是装饰。
- 移动端首屏不过度拥挤。
