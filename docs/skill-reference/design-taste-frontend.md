# design-taste-frontend

## 适用场景

用于对已有页面或即将实现的页面做审美校准。它像一个前端审美和工程质量 gate，重点处理 LLM 生成 UI 的统计偏差。

## 输入

- 已有页面截图、URL 或源码。
- 页面目标和目标用户。
- 期望视觉密度。
- 动效强度。
- 技术栈和组件库限制。

## 输出

- 反 AI 味问题清单。
- 设计修正原则。
- 组件、布局、状态和响应式要求。
- 必要时输出修改后的代码。

## 校准维度

### 1. 排版

- 技术工具页优先高级 sans-serif 或 mono 搭配。
- 创意/editorial 页面可以使用更有性格的 serif 或 display 字体。
- 标题不要只靠巨大字号建立层级，应配合字重、颜色和位置。

### 2. 色彩

- 最多一个主要强调色。
- 降低过饱和颜色。
- 禁止无意义的紫蓝霓虹渐变。
- 同一页面不要混用冷灰和暖灰。

### 3. 布局

- 避免默认居中 hero。
- 避免三列等宽卡片作为每个 section 的默认答案。
- 高密度页面用线、分组和负空间，不要全靠卡片盒子。
- 非对称布局在移动端必须回退为稳定单列。

### 4. 状态

页面或组件至少要考虑：

- Loading：使用骨架而不是随便一个 spinner。
- Empty：告诉用户如何产生内容。
- Error：错误位置清晰，不只弹 toast。
- Active/Pressed：按钮和可点击元素有触感反馈。

### 5. 性能

- 动画只用 transform 和 opacity。
- 不动画 top、left、width、height。
- 背景纹理不要放在滚动容器里持续重绘。
- 不滥用高 z-index。

## 检查清单

- 是否用了默认 AI 配色？
- 是否所有内容都被卡片包住？
- 是否有真实的交互状态？
- 移动端是否横向溢出？
- 字体和内容气质是否匹配？
- 有没有无法验证的数据、客户或奖项？
