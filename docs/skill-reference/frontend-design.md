# frontend-design

## 适用场景

用于生成或重做用户可见的前端页面：网站、landing page、作品集、dashboard、HTML/CSS 原型、React/Vue 组件等。

它解决的问题是：模型直接收到“做得好看点”时容易回到平均模板。这个参考要求先确定鲜明但合理的设计方向，再写代码。

## 输入

- 页面目标和目标用户。
- 内容优先级。
- 品牌或产品气质。
- 技术栈和约束。
- 参考站点或截图。
- 禁止暴露的信息。

## 输出

- 视觉方向说明。
- 字体、颜色、空间、组件和动效策略。
- 可运行的前端代码。
- 桌面和移动端检查结果。

## 核心方法

1. 先选方向。
   - 例如编辑感、工业风、极简工具、产品档案、战报、控制台、奢华、玩具感。
   - 方向必须服务内容，不是为了炫技。

2. 再定系统。
   - 字体：避免默认字体堆叠，选择和气质一致的 display/body 搭配。
   - 色彩：限制主色和强调色，避免一页多个互相竞争的 palette。
   - 空间：决定是高密度工具页还是更有留白的叙事页。
   - 组件：按钮、标签、卡片、表格、时间线必须统一。

3. 最后实现。
   - 只修改真实源码入口。
   - 不直接改构建产物。
   - 为固定格式元素定义稳定尺寸，避免 hover 或动态文本撑坏布局。
   - 设计移动端优先级，而不是简单缩小桌面版。

## 反模板要求

避免这些常见 AI 前端痕迹：

- 紫蓝渐变。
- 居中大标题加三列卡片。
- 默认 Inter/Arial/system 字体。
- 所有模块都套卡片。
- 过大的圆角和阴影。
- 只有成功态，没有 loading、empty、error、active 状态。
- 移动端横向滚动或文字挤出按钮。

## 验收标准

- 页面一眼能看出属于这个主题，而不是通用模板。
- 字体、颜色、间距和组件风格一致。
- 视觉强度没有压过内容真实性。
- 桌面和移动端都经过截图检查。
- 没有假数据、假客户、假推荐语和隐私泄露。
