# design-shotgun

## 适用场景

当视觉方向还不确定时使用。不要在一个坏方向上反复微调，而是先快速生成几套差异明显的方向，再选一个继续深化。

## 输入

- 页面或产品 brief。
- 目标用户。
- 必须保留的信息。
- 明确不想要的风格。
- 参考站点或竞品截图。

## 输出

- 3-5 个视觉方向。
- 每个方向的设计意图。
- 方向之间的差异说明。
- 对比板或截图集合。
- 选择方向后的下一步实现建议。

## 工作流

1. 收集上下文。
   - 页面目标是什么？
   - 用户进来第一眼要理解什么？
   - 内容有什么真实性边界？

2. 定义方向坐标。
   - 例如：严肃/活泼、密集/留白、产品感/编辑感、保守/实验。
   - 每个方向必须在坐标上有明显差异。

3. 生成方向。
   - 不要只换颜色。
   - 每个方向要改变信息结构、排版节奏、视觉材料和组件语言。

4. 对比并收敛。
   - 哪个最符合目标用户？
   - 哪个最能承载真实内容？
   - 哪个最不容易变成模板？
   - 哪个实现成本和维护成本可接受？

5. 锁定方向后再写代码。

## 方向卡模板

```text
方向名称：
适合原因：
视觉关键词：
字体策略：
色彩策略：
布局策略：
适合的内容：
风险：
不适合的情况：
```

## 验收标准

- 至少 3 个方向真的不同，而不只是换色。
- 每个方向都能解释取舍。
- 先选择方向，再进入组件和代码细化。
- 不复制参考站点的资产、logo、图片和专属文案。
