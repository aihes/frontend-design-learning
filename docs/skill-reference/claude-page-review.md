# claude-page-review

## 适用场景

用于用户可见页面发布前的第二模型审查，尤其是首页、关于页、定价页、文档页、onboarding、landing page 和作品集页面。

## 输入

- 页面 URL、截图或源码。
- 页面目标。
- 目标用户。
- 已有设计系统。
- 不能暴露的信息。
- 允许修改的文件范围。

## 输出

- 页面问题清单。
- 改稿 brief。
- 文案和层级建议。
- 真实性和隐私检查。
- 发布前验证清单。

## 工作流

1. Codex 先定义问题。
   - 页面哪里不清楚？
   - 是文案、层级、视觉、交互还是真实性问题？

2. 写 bounded brief。
   - 页面路径。
   - 业务目标。
   - 必须保留的事实。
   - 禁止暴露的信息。
   - 允许改哪些文件。

3. 让第二模型审查。
   - 重点看用户是否能理解。
   - 不让第二模型自由编造客户、数据或产品能力。

4. Codex 回收结果并验证。
   - diff 是否合理。
   - 文案是否更清晰。
   - 是否引入假信息。
   - 是否符合设计系统。
   - 浏览器和移动端检查是否通过。

## 审查重点

- 首屏是否回答用户真正关心的问题。
- 页面是否暴露内部实现、账号、路径或不可公开信息。
- 文案是否有空泛营销词。
- 指标、客户、奖项、认证是否可验证。
- CTA 是否明确。
- 移动端是否可读。

## 验收标准

- 第二模型只做页面审查，不接管整个工程。
- 改稿范围明确。
- 不新增不可验证的声明。
- 发布前有截图或浏览器验证。
- 隐私和品牌风险被显式检查。
